# 🎯 招聘助手 - HireHelper

一款专为招聘网站设计的浏览器插件，根据公司介绍自动生成个性化打招呼话术，靠前位置突出你的核心优势与岗位匹配点。

## 💡 为什么做这个插件

做这个插件的初衷是为了打破找工作时的信息壁垒。HR 在招聘平台上看到的求职者开头语往往决定了你是否能引起注意，但短短几个字很难展示你的全部能力。所以需要在靠前位置就突出你的核心优势与岗位的匹配点，让 HR 一眼看到你的价值。

本插件会综合读取以下信息，让 AI 为你生成有针对性的打招呼话术：

- 📋 **公司简介** — 自动识别当前招聘页面的公司信息
- 📝 **岗位要求** — 提取职位描述中的关键需求
- 📁 **skills/SKILL.md** — 你的个人能力档案（技术栈、学历、亮点经历）
- 📁 **skills/RESUME.md** — 你的完整简历（可选，更丰富的人岗匹配上下文）
- 📁 **rules/RULES.md** — 你对 AI 输出风格的约束规则 + 示例话术

希望这个小工具能让你的求职之路轻松一点 🤝

## ✨ 功能特性

| 功能 | 说明 |
|------|------|
| 🔍 **智能识别** | 点击插件自动识别当前招聘页面的公司信息 |
| 📸 **截图识别** | 支持截图后自动提取页面信息 |
| 📋 **粘贴文本** | 支持手动粘贴公司介绍/职位描述 |
| 🤖 **AI 增强** | 接入 OpenAI 兼容 API，生成更智能、个性化的话术 |
| ⚡ **流式输出** | SSE 逐条展示话术，首条 ~1-2 秒即出，带打字动画 |
| 🎯 **精准匹配** | 自动匹配你的技能与公司需求，靠前位置突出优势 |
| ⚙️ **能力配置** | 设置你的技能标签、亮点描述、自定义模板 |
| 📝 **示例引导** | 从 `rules/RULES.md` 自动提取示例话术作为 few-shot 引导 |

## 🚀 安装方法

### 1. 克隆项目

```bash
git clone <repo-url>
cd hire-helper
```

### 2. 创建私有配置文件

项目克隆后，你需要手动创建以下 **3 个文件/文件夹**（已 git 忽略，不会被提交）：

#### ① `ai-config.js` — AI 接口配置

在项目根目录创建 `ai-config.js`：

```js
// ========================================
// AI 配置文件 — 请勿提交到 Git
// ========================================
//
// ⚠️ 配置优先级：扩展设置页保存的配置会覆盖本文件。
// 若改本文件后仍 401，请到设置页重新保存，或清除 chrome.storage.local 中的 aiConfig。

const AI_CONFIG = {
  // API 地址（支持 OpenAI 及兼容接口：DeepSeek、通义千问、智谱等）
  baseUrl: 'https://api.openai.com/v1',

  // 模型名称
  model: 'gpt-4o-mini',

  // API Key（必填）
  apiKey: 'sk-your-api-key-here',

  // 生成参数
  temperature: 0.8,
  // 推理模型（如 mimo-v2.5-pro）需要更大的 max_tokens，否则思考占满配额导致 content 为空
  maxTokens: 2048,

  // 系统提示词（角色定义；具体输出规则见 rules/RULES.md，调用时自动注入）
  systemPrompt: `你是招聘话术助手。根据目标公司信息和个人档案，生成个性化的打招呼话术。`,
};
```

#### ② `skills/` — 个人能力档案

创建文件夹和文件 `skills/SKILL.md`（必须）和 `skills/RESUME.md`（可选）：

```markdown
# 个人能力档案

> 修改此文件，填写你的个人信息，AI 生成话术时会自动携带。

---

## 基本信息

3 年前端开发经验，计算机科学本科

## 技术栈

React, Vue3, TypeScript, Node.js, PostgreSQL

## 亮点经历

主导过日活百万级小程序开发，有开源项目 star 2k+

## 求职意向

期望岗位：高级前端工程师 / 目标城市：北京 / 期望薪资：30-40k
```

> 💡 `skills/RESUME.md` 为可选的完整简历文件，内容会作为补充上下文注入 prompt。

#### ③ `rules/` — AI 输出规则 + 示例话术

创建文件夹和文件 `rules/RULES.md`：

```markdown
# AI 输出规则

> 定义 AI 生成话术时必须遵守的约束，可自由增删。

---

## 格式要求

- 生成 5-8 条不同风格的话术
- 每条话术用换行分隔，不要编号、不要引号
- 不限制总字数，但靠前位置必须突出核心优势或岗位匹配点
- 参考下方示例话术的风格和长度

## 风格要求

- 语气自然真诚，像真人对话，不要像模板
- 不要用「！！」等过度感叹
- 可以适当口语化，但不要出现错别字

## 称呼规范

- 默认开头用「您好」，后面直接接优势或岗位匹配点
- 优先根据目标公司信息中的招聘人/联系人信息选择更合适的称呼
- 不要编造招聘人姓名或职级

## 内容要求

- 必须结合 SKILL.md 中的个人能力来写，体现人岗匹配
- 必须提及或呼应目标公司的特点
- 不要夸大，不要编造不存在的经历
- 不要出现薪资、年龄、性别等敏感信息

## 禁止事项

- ❌ 不要输出除话术以外的任何内容
- ❌ 不要使用「作为 AI」「作为语言模型」等表述
- ❌ 不要输出 markdown 格式，纯文本即可

## 示例

- 您好，我有 8 年前端经验，现任前端组长，熟悉 React + TypeScript，与贵司岗位较匹配，方便发简历详聊吗？
- 老师您好，8 年前端开发，2 年带组经验，主导过金融 AI 平台落地，看到贵司招前端，想进一步沟通。
- 您好，前端组长背景，擅长 B 端中后台与 SDK 工程化，感觉与贵司业务方向契合，可否聊聊？
```

> 💡 `## 示例` 区块中的话术会自动提取为 few-shot 示例，注入 AI prompt 引导生成风格。示例中的 `【】` 会被模型理解为需要替换为实际岗位信息的占位符。

### 3. 加载插件

1. 打开 Chrome 浏览器，地址栏输入 `chrome://extensions/`
2. 右上角开启 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择 `hire-helper` 文件夹
5. 插件图标出现在浏览器工具栏 🎉

## 📖 使用方法

### 第一步：配置你的能力

1. 点击插件图标
2. 点击右上角 ⚙️ 进入设置
3. 添加你的 **技能标签**（如: React, Python, AI, Go...）
4. 添加你的 **亮点描述**（如: 3年大厂经验, 有开源项目...）
5. 可选：添加 **自定义模板**（支持占位符）
6. 保存

> 💡 `skills/SKILL.md` 中填写的档案会作为 AI 的核心上下文，建议认真填写。

### 第二步：开启 AI 增强

1. 在主界面打开 **🤖 AI 增强** 开关
2. 确认状态显示「已配置」（需在 `ai-config.js` 或设置页填写 API Key）

> ⚠️ **配置优先级**：设置页保存的 AI 配置会覆盖 `ai-config.js`。若只改了文件却报 401，请到设置页重新保存 API Key，或清除旧配置后重新加载扩展（见下方「常见问题」）。

### 第三步：生成话术

1. 打开任意招聘网站的职位页面（BOSS直聘、拉勾、猎聘、智联等）
2. 点击插件图标
3. 点击 **🔍 识别页面信息**
4. 查看生成的话术，点击即可复制

### 其他方式

- **📸 截图识别**：截取页面后粘贴文本识别
- **📋 粘贴文本**：手动粘贴公司介绍或职位描述
- **🔄 换一批**：刷新生成更多话术（会自动取消上一次正在进行的流式生成）

> 关闭 AI 开关后，将使用本地模板引擎生成话术，无需网络。

## 🤖 AI 接入说明

本插件支持 OpenAI 兼容接口，可接入以下服务：

| 服务 | baseUrl | model |
|------|---------|-------|
| OpenAI | `https://api.openai.com/v1` | `gpt-4o-mini` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` |
| 智谱 | `https://open.bigmodel.cn/api/paas/v4` | `glm-4-flash` |

只需修改 `ai-config.js` 中的 `baseUrl`、`model`、`apiKey` 即可切换。也可以在扩展 **设置页** 直接填写，设置页的值优先级更高。

### 配置优先级

扩展读取 AI 配置的顺序如下（前者覆盖后者）：

1. **设置页**（保存到 `chrome.storage.local` 的 `aiConfig`）
2. **`ai-config.js`**（本地文件，作为默认值）

因此，如果你曾在设置页保存过 API Key，之后只改 `ai-config.js` **不会生效**，请求仍会使用设置页里的旧 Key，从而出现 `401 Invalid API Key`。

**推荐做法：**

- 首次配置：填写 `ai-config.js`，或在设置页填写并保存（二选一即可）
- 更换 Key / 接口：到设置页更新并保存；或清除旧配置后改文件

**清除设置页覆盖的配置**（在 `chrome://extensions` 打开本扩展的 Service Worker 控制台）：

```js
chrome.storage.local.remove('aiConfig')
```

执行后重新加载扩展，配置会回退到 `ai-config.js`。

### 常见问题：401 Invalid API Key

| 原因 | 解决办法 |
|------|----------|
| 设置页旧 Key 覆盖了 `ai-config.js` | 到设置页重新粘贴并保存，或执行上方 `remove('aiConfig')` |
| API Key 复制不完整（多/少字符） | 从服务商控制台完整复制，注意首尾无空格 |
| 改完 `ai-config.js` 未重载扩展 | 到 `chrome://extensions` 点击「重新加载」 |
| Base URL 与协议不匹配 | OpenAI 兼容接口用 `.../v1`，Anthropic 兼容用服务商提供的 anthropic 地址 |

**AI 调用流程：**

```
点击识别 → 抓取页面公司信息
         → 读取 skills/SKILL.md + RESUME.md（个人档案，缓存 5 分钟）
         → 读取 rules/RULES.md（输出规则 + 示例话术，缓存 5 分钟）
         → 组装 prompt（示例 few-shot + 个人档案 + 公司信息）
         → SSE 流式调用 AI API
         → 逐条返回话术（带打字动画）
```

## 🧩 话术生成原理

### 本地模式（AI 关闭）

1. **关键词提取**：从公司介绍/职位描述中提取技术关键词
2. **技能匹配**：将你的技能标签与公司需求进行匹配
3. **优势组合**：优先使用你的亮点描述，其次使用技能匹配的优势
4. **模板填充**：按预设模板生成话术
5. **去重排序**：过滤重复话术，按匹配度排序展示

### AI 模式（AI 开启）

1. **信息抓取**：自动识别页面公司信息
2. **上下文注入**：读取 `skills/SKILL.md` + `skills/RESUME.md` + `rules/RULES.md`（自动缓存，5 分钟过期）
3. **示例提取**：从 `rules/RULES.md` 的 `## 示例` 区块自动提取 few-shot 话术
4. **Prompt 组装**：将示例、个人档案、输出规则、公司信息组装为结构化 prompt
5. **流式生成**：调用 OpenAI 兼容 API（SSE 流式），逐条返回话术，首条约 1-2 秒即出
6. **智能降级**：AI 调用失败时自动回退到本地模板引擎
